Unity - Official Tutorials

Or go back to home

I recommend you do some of Unity's official tutorials.

If you find the videos too fast to keep up (or too slow and boring), you can change the video speed by clicking on the cog icon in the lower right hand corner of the video player.

Here are some extra notes about some of the tutorials. All times given are in hours and minutes, so 1:15 means 1 hour, 15 minutes. I hope this helps you decide which tutorials to spend your time on.

Roll-a-ball tutorial (1:15)

Short and simple. Everyone should do this.

Highlights:

  • Moving the camera
  • Displaying score on the screen
  • Make something happen when the player touches an object

Space Shooter tutorial (5:06)

Split into 3 hours of basics, then 2 hours of extension work (which I recommend doing). There's also 1:36 on making a mobile (touch screen) version which I didn't watch and didn't count in the total time.

Warning: the video was made using an older version of Unity. Messages will pop up telling you when you have to do something different to how it is shown. If you are still confused, click on the 'YouTube' link in the lower right corner of the video to go to YouTube, and look through the comments to see if someone has helpful advice.

Highlights:

  • better lighting
  • shooting \ making shots appear
  • using a game controller to make waves of enemies appear
  • using arrays to easily have variations of enemies (or anything)
  • explosion effects
  • playing sounds

Survival Shooter tutorial (3:55)

(Only 3:35 if you leave out the final 20 minutes that are about special audio features.)

Warning: the video was made using an older version of Unity. The download includes a document listing some of the changes you will have to make, so glance through that before you start. If you still get stuck, click on the 'YouTube' link in the lower right corner of the video to go to YouTube, and look through the comments to see if someone has helpful advice.

I encountered a bug with the damage puff emitter. If you shot an enemy, waited, then shot again, the damage effect didn't appear the second time or ever again. https://issuetracker.unity3d.com/issues/particle-system-plays-only-once?page=2#comments - see JEFEQUESO's solution.

Highlights:

  • Using a game controller to create waves of enemies (again)
  • displaying score (again)
  • Enemy movement AI with the navmesh
  • Animating characters with mechanim and triggers
  • Make a healthbar (using a UI slider, sneaky)
  • Make the screen flash when you are hurt
  • Shooting (raycasts), impact effects
  • Control what a shot hits and what it passes through. Layer masks and LayerMask.GetMask("mask name")
  • Health and damage, enemies bite when close to the player
  • Animated UI (making text change size, fade in and out, etc, using mechanim)
  • UI elements are drawn in layers in the order they appear in the heirachy view
  • Fade the camera to black (or any color) using a full-screen UI image - part 10
  • You can animate anything using mechanim! positions, shapes, sizes, colors!

Small but interesting tips:

  • SetActive is for a whole object, enabled is for one component. - part 7
  • Apply changes to a prefab (button at top of inspector)
  • Give a gameobject a custom icon (top left in the Inspector)
  • You can edit a script by clicking on its name in the 'script' field in inspector.
  • You can comment with /* */ as well as //

Tanks tutorial (3:55)

tanks phase 1

  • use simple colliders on everything, not mesh colliders
  • dust trails: a particle emitter can emit particles over distance
  • (set simulation space to world and emission on distance)
  • Game Objects have void OnEnable() as well as Awake() and Start(). OnEnable happens every time it's enabled, great for resetting.
  • On a rigidbody, kinematic means 'forces do not affect it'
  • autocomplete is helpful

tanks phase 2

  • Rotations and moving things
  • Finding errors, especially unassigned reference errors
  • changing audio pitch to avoid phasing, audio changing with speed
  • using prefabs, updating, seeing what has changed from the prefab

tanks phase 3 - camera

  • camera rig to help position the camera
  • orthographic cameras, how they work in detail
  • Vector3.SmoothDamp
  • Move camera to show everyone in a group of objects!!

tanks phase 4

  • putting UI in the game world instead of screen (a few settings to change)
  • (make a canvas a child object of a game object to make it move around with it)
  • Make a circular\radial health bar
  • Performance tip: Reuse a particle effect instead of creating a new one each time
  • (handy tip: hold alt and expand something in the Heirachy to expand the whole tree)
  • (handy tip: hold shift and select multiple things in the heirachy. You can edit multiple objects at once)

tanks phase 5 - Shell Creation

  • (handy tip: You can call destroy with a delay, for example "destroy this shell in 5 seconds")
  • Deal damage and apply movement force in an explosion area using OverlapSphere
